**# O2A — Outlook til Aula**

**## Hvad er O2A?**
To afsnit om kernefunktionen: O2A er et Windows-desktopprogram der automatisk holder Outlook-aftaler mærket med kategorien "AULA" synkroniseret med Aula-kalenderen. Kerneværdien: Outlook-aftaler mærket "AULA" afspejler altid den korrekte tilstand i Aula — automatisk, uden manuel indgriben.

**## Sådan virker det**
Nummereret liste (baseret på ARCHITECTURE.md Data Flow):
1. Brugeren klikker "Kør O2A" eller timeren udløser automatisk kørsel (op til hver 4. time)
2. Programmet logger ind på Aula via UNI-login HTML-formular (automatisk login ved hjælp af gemt brugernavn og adgangskode)
3. Outlook-kalender læses via Windows COM-integration — kun aftaler med kategorien "AULA" eller "AULA Institutionskalender" hentes
4. Aula-kalender hentes via Aulas REST API (v23) — programmet genkender egne begivenheder via skjulte mærker i begivenhedsbeskrivelserne (o2a_outlook_GlobalAppointmentID)
5. De to kalendere sammenlignes: nye aftaler oprettes i Aula, ændrede aftaler opdateres, slettede aftaler fjernes
6. Eventuelle fejl vises i GUI-logpanelet og sendes som fejlmail via Outlook

**## Hvad sker der ved fejl?**
Kort forklaring: Fejl logges i o2a.log og vises i GUI-panelet. Ved fejl ved oprettelse/opdatering sendes en HTML-fejlmail til brugeren via Outlook. Login-fejl afsluttes med fejlstatus.

**## Begrænsninger**
Bullet-liste:
- Kun Windows (kræver Microsoft Outlook og Windows COM-integration)
- Tilbagevendende begivenheder (f.eks. ugentlige møder) synkroniseres IKKE korrekt — de oprettes som enkeltbegivenheder. Ingen advarsel vises i programmet i dag (kendt problem)
- Ingen tovejs-synkronisering: Ændringer direkte i Aula overskrives ved næste sync
- Programmet synkroniserer kun frem til ca. 1 år frem fra i dag
- Kun aftaler med Outlook-kategorien "AULA" synkroniseres

**## Mappestruktur (aktive filer)**
Forenklet tabel med kolonnerne Fil/mappe og Formål — kun aktive filer baseret på STRUCTURE.md:
- main.pyw — Programmets startpunkt og synkroniseringslogik
- aula/ — Pakke med Aula API-klient (aula_calendar.py er den aktive kode)
- outlookmanager.py — Læser Outlook-kalender via Windows COM
- calendar_comparer.py — Sammenligner de to kalendere og finder forskelle
- setupmanager.py — Håndterer konfiguration og bruger-login-setup
- peoplecsvmanager.py — Mapper Outlook-navne til Aula-navne via personer.csv
- configuration.ini — Brugerkonfiguration (brugernavn, indstillinger)
- personer.csv — Navne-alias-liste (Outlook-navn → Aula-navn)
- personer_ignorer.csv — Navne der skal ignoreres ved synkronisering
- o2a.log — Logfil (oprettes automatisk ved kørsel)

Bemærk: aulamanager.py, eventmanager.py og databasemanager.py eksisterer i kodebasen men er dødkode — de bruges ikke af den aktive synkronisering.

**## Teknologi**
Simpel liste baseret på STACK.md (med forklaring i parentes ved tekniske termer):
- Python 3 — Programmeringssprog
- PySide6 — GUI-framework (Qt6; bruges til vindue, systembakke-ikon og dialoger)
- pywin32 — Windows-integration (bruges til at tale med Microsoft Outlook)

- keyring — Sikker opbevaring af adgangskode i Windows Credential Manager
- configuration.ini — Brugerkonfiguration i tekstformat
